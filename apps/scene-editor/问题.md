
# 遇到的问题
1. 用户导入模式时，模型内容中存在多级层级，真正的模型在最内层，逐层选择很麻烦
去除掉多余的层级，将模型内容放到根目录下。没有使用 threejs 自带的 flatten 方法，因为直接使用这个方法，会使得内部某些部件相对位置不对。
使用自定义的 bakeFlatten 方法，把世界矩阵直接作用到顶点上。顶点是点集,任意仿射矩阵(含切变)作用其上都是精确的,因此相对位置不会丢失。

2. 用户导入自定义的模型，但是单位不同，尺寸会有很大差别
进行启发式自动修正，首先根据模型的最长边来大致判断模型的单位是什么
- 最长边 ≤ 10m(在带内或偏小)→ ×1 不动(偏小更可能是合理小物件,不冒险放大);
- 偏大 → 在 cm/mm 公制因子里选换算后最接近 1m 的 → 缩小到米制
- FBX格式，头部带 UnitScaleFactor（通常 cm），转换时一般已被应用，可以不被转化

3. 旋转场景，活动的范围很大，即使选中某个物体也是这样
封装 Three.js 的 OrbitControls 控制器（OrbitCameraControl.ts），自行实现右键点击时以被指向的模型为中心旋转场景  
自行实现之前，主要的问题是：为了将右键指向的模型作为旋转场景的中心点，用右键点击来选中没有在视图中间的模型时，相机会自动指向这个模型而导致视图跳动。
**原因：**
原生 OrbitControls 强制相机始终"看向" target，而 target 既是旋转中心、又是视线中心——这两个角色被绑死了。并且它每帧都会执行`camera.lookAt(target)`，将相机镜头对准target

**封装的逻辑：**
- 右键交给自定义逻辑（mouseButtons.RIGHT = null），左键平移、中键缩放保持不变
- 右键拖拽时绕轴心旋转：同时旋转相机的位置（position 绕轴心）和朝向（quaternion 叠加同一旋转），使轴心点在屏幕上保持原位 → 不再跳动
- 旋转期间 update() 跳过 orbitControls.update()，避免其 lookAt(target) 把朝向拉回；松开右键后把 target 重新对齐到视线方向，让 OrbitControls 平滑接管
- 俯仰角做了极角夹紧，防止翻转
- 自定义旋转进行时跳过 OrbitControls 更新，避免其 lookAt(target) 覆盖朝向，将“看向哪里”和“绕哪里转”分开
